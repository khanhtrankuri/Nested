📈 Architecture Selection Logic

graph TD START["Create AdvancedPolyMemnet"] START --> EN_EN["enable_nested?"] EN_EN -->|No| STD["Standard Encoder
no modifications"] STD --> STD_DEC["Decoder?"] EN_EN -->|Yes| CMS_ENC["CMSSelfModifyingEncoder"] CMS_ENC --> USE_MOD["Use modifications?"] USE_MOD -->|No| STD_ENC["Standard backbone

cross-stage + adaptive + meta"] STD_DEC -->|use_advanced_decoder| CMS_DEC["CMSDecoder"] STD_DEC -->|else| FPN_DEC["FPNDecoder + Refiner"] CMS_DEC --> OUT["Output"] FPN_DEC --> OUT FULL --> CMS_DEC STD_ENC --> FPN_DEC
Key Points:
• enable_nested=True mới kích hoạt hệ thống nested learning (prototypes, uncertainty, enhanced loss)
• use_advanced_decoder=True → dùng CMSDecoder (thay thế cả decoder + refiner)
• use_advanced_decoder=False + enable_nested=True → FPNDecoder + SafeNestedResidualRefiner
• Encoder modifications (cross-stage, adaptive, meta) chỉ hoạt động khi enable_nested=True và respective flags
